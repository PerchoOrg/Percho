一、背景与约束

产品定位

Vicinity V1 是面向 listing agent 的内容工具,服务全美购房者,不是双边平台。三条核心链路:

  • Listing 展示页 — 公开可分享,视频优先,移动端沉浸式
  • Agent Dashboard — 创建/编辑 listing,查看数据,收 lead 通知
  • Lead 流转 — 100% 归 agent,平台不分流

技术约束(来自 PRD v10)

  • Agent 完成一个 listing 上传到发布 ≤ 30 分钟(UX 硬约束)
  • 视频:Agent 自己拍摄,直传平台
  • 规模:V1 支持 10–50 个 agent 并发
  • Fair Housing 合规:严禁展示种族/族裔构成数据
  • 社区数据:V1 手动录入,强制带 source_url + recorded_by + recorded_at
  • 语言:英文单语,V1 不做双语,不做小红书 / 微信渠道
  • 团队:V1 一人 + Claude;架构需 scale 到后期扩团队,不写 throwaway
定位重申:面向全美所有购房者,不针对特定族裔。所有面向用户的文案 / schema / 第三方渠道选型,都不假设族群定位。

二、技术选型

总原则

零 DevOps · 一人 + Claude 4 周上线 · 数据层不锁死(Postgres + 标准 SQL,日后切 RDS/Aurora 一周工作量)。

层级选型理由
前端框架Next.js 14 (App Router)同一 codebase 同时跑 SSR(SEO listing 页)+ CSR(dashboard)
样式Tailwind CSSDemo 已用,保持一致
动效Framer MotionDemo 已用,竖屏卡片动画
托管Vercelpush 即部署,自动 HTTPS,全球边缘
BaaSSupabasePostgres + Auth + Storage + Edge Functions 一体,免去 4 个独立选型
视频Cloudflare StreamTUS 续传 + 自动 HLS 转码 + 全球 CDN,按分钟计费
邮件ResendLead 通知,5 分钟集成,$0/月起
地图Google Maps JS API通勤路线可视化是 listing 页核心差异化
LLMAnthropic API directlisting 描述生成,async 调用,失败重试
分析Vercel Analytics + 自定义事件第一天就埋 page_view / card_view / lead_submit

关键选型说明

为什么 Supabase 而不是自建 RDS

RLS 在数据库层直接保证 agent 只能看自己的 listing,免去应用层权限代码,更安全。Migrations 是标准 SQL git-managed,日后切 RDS 是改 connection string,不是重写。

为什么 Cloudflare Stream 而不是 Supabase Storage 直传

Agent 上传的是手机原视频,可能 4K 1GB+,需要 TUS 续传(手机网络不稳)、自动多码率 HLS 转码、全球 CDN。Supabase Storage 是对象存储,搞不定这些。

为什么 Vercel 而不是 ECS Fargate

一人 + Claude 用 Vercel 4 周能上线,Fargate 同样工作量 6–8 周。代价是 vendor lock —— 但 Next.js 是开源,真要切 self-host(Docker + ALB)是 1–2 周不是重写,这个 tech debt 可接受。

RLS 策略(数据库层权限护栏)

-- agent 只能看到自己的 listing
alter table listings enable row level security;
create policy "agent sees own listings"
  on listings for all
  using (agent_id = (select id from agents where user_id = auth.uid()));

-- public 可读已发布的 listing
create policy "public reads published listings"
  on listings for select
  using (status = 'published');

-- agent 只看自己的 lead
alter table leads enable row level security;
create policy "agent sees own leads"
  on leads for all
  using (agent_id = (select id from agents where user_id = auth.uid()));

五、视频上传流程

Dashboard 视频上传是核心体验,关键点:浏览器直传 Cloudflare,绕开 Vercel(后者有 4.5MB body 上限,大视频根本传不过去)。

Agent 选视频
   │
   ▼
前端调用  POST /api/video/create-upload
   │
   ▼
API Route → CF Stream 创建 TUS 直传 URL
   │            返回 upload_url + cf_video_id
   ▼
浏览器用 tus-js-client 直接上传到 Cloudflare
   │  (断点续传 · 进度条 · 不经 Vercel)
   ▼
CF Stream 自动转码多码率 HLS
   │
   ▼
转码完成触发 webhook → Supabase Edge Function
   │  videos.status = 'ready'
   ▼
Supabase Realtime 推送 Dashboard
   │  WebSocket 实时更新 UI
   ▼
Buyer 端 listing 页直接播 HLS manifest
   (Vercel 整条链路只参与签发 upload URL,不背视频流量)

关键代码片段

// app/api/video/create-upload/route.ts
export async function POST(req: Request) {
  const session = await getServerSession();
  if (!session) return new Response('Unauthorized', { status: 401 });

  const { listingId, type } = await req.json();

  const cfRes = await fetch(
    `https://api.cloudflare.com/client/v4/accounts/${CF_ACCOUNT_ID}/stream?direct_user=true`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${CF_API_TOKEN}`,
        'Tus-Resumable': '1.0.0',
        'Upload-Length': req.headers.get('upload-length') || '0',
        'Upload-Metadata': `maxDurationSeconds 300`,
      }
    }
  );

  const cfVideoId = cfRes.headers.get('stream-media-id');
  const uploadUrl = cfRes.headers.get('location');

  await supabase.from('videos').insert({
    listing_id: listingId,
    cf_video_id: cfVideoId,
    type,
    status: 'processing',
  });

  return Response.json({ uploadUrl, cfVideoId });
}

文案生成 API

// app/api/generate-copy/route.ts
export async function POST(req: Request) {
  const { listing } = await req.json();

  const response = await anthropic.messages.create({
    model: 'claude-opus-4-7',
    max_tokens: 800,
    messages: [{
      role: 'user',
      content: `Write a 3-paragraph English listing description.
        Address: ${listing.address}, ${listing.neighborhood}
        Price: $${listing.price.toLocaleString()}
        Layout: ${listing.beds} bed ${listing.baths} bath ${listing.sqft} sqft
        Style: ${listing.style}

        Return JSON: { "description": ["para1", "para2", "para3"] }`
    }],
  });

  return Response.json(JSON.parse(response.content[0].text));
}

九、安全与合规

Fair Housing 合规

  • Schema 层不存任何种族/族裔比例字段(table 设计中已排除)
  • Listing 页 UI 不展示任何人口构成数据
  • 允许展示:学区评分 / 收入中位数(county 级别)/ 教育水平 / 犯罪指数 / 年龄结构

数据源审计

每条社区数据(schools / pois)三个字段强制非空:

  • source_url(必填,DB 约束)
  • recorded_by(必填,DB 约束)
  • recorded_at(自动时间戳)

Dashboard 编辑界面显式要求填写数据源 URL,提交时校验非空。

视频内容审核(V1 简版)

  • 上传时限制文件类型(video/*)和大小(≤ 2GB)
  • Cloudflare Stream 自动违规扫描
  • 手动审核前不允许 publish
